// 1. Leer configuración del CPANEL
const savedEnv = localStorage.getItem('APP_ENV') || 'dev';

const prodUrl = localStorage.getItem('APP_PROD_URL') || 'https://api.tu-app.com';

// 2. Asignar URL dinámica

PROTOCOLO DE RESCATE: INSTALACIÓN DESDE CERO
¿Base de datos corrupta? ¿No arranca? Sigue estos pasos exactos en tu terminal CMD o Cmder para reconstruir todo el ecosistema.
FASE 1: LIMPIEZA NUCLEAR ☢️

Cierra todas las consolas abiertas. Ve a la carpeta de tu proyecto y elimina la base de datos corrupta:

rmdir /s /q pb_data
del pb_migrations\*.js

* Nota: Esto borrará TODOS los usuarios y registros. Solo hazlo si quieres empezar de cero.

FASE 2: REINSTALAR POCKETBASE 📦

Abre tu CMD o Cmder en la carpeta del proyecto y ejecuta el servidor de nuevo. Esto creará una base de datos virgen.

.\pocketbase.exe serve
Crear el Superusuario Master:
  1. Abre tu navegador y entra a: http://127.0.0.1:8090/_/
  2. Te pedirá crear el primer administrador.
  3. Pon tu email master y una contraseña segura.
  4. ¡Listo! Tu BD ha revivido. Cierra la consola con Ctrl+C para seguir al siguiente paso.
FASE 3: INICIAR EL ORQUESTADOR Y TÚNEL 🌐

Para arrancar PocketBase + Tailscale juntos automáticamente:

npm run start-global

* Minimiza esta consola. El servidor está vivo y publicado en internet.

FASE 4: SINCRONIZAR Y COMPILAR LA APP (CAPACITOR) 📱

Abre OTRA consola nueva en la carpeta de tu proyecto. Inyecta el código al contenedor nativo:

npx cap sync

Luego, entra a la carpeta Android, limpia la caché vieja y compila el APK final:

cd android
gradlew clean
construir.bat
¡Sistema Restaurado! Ahora vuelve al CPanel y prueba el Login de nuevo.
